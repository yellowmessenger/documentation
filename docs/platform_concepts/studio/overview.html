Studio overview

The studio provides a no-code bot-building platform that helps anyone create a bot for their business needs easily. With Studio, you can collect information from your customers, update their information, launch actions, show them the necessary information, have a live chat with an agent and do a lot more.

This article briefs you on understanding various components of Studio and helps you to get started with creating a bot.

1. Access studio​

info

Before designing your bot, you need to have the bot set up as mentioned here.

  1. Log in to https://cloud.yellow.ai.
  2. On the switcher, click Studio.

tip

You can access Studio using the keyboard shortcut g + s.

2. Navigate the Studio homepage​

The studio homepage showcases a collection of created and imported flows. It offers the following options to enhance your bot-building experience:

  1. Conversation Settings: Customize the way your bot engages with end users.
  2. Preview: Take a sneak peek at how your bot will interact before deploying it.
  3. Publish changes: Make your bot live and unleash its full potential.
  4. Welcome (Start flow): The initial flow users encounter when interacting with the bot.
  5. Fallback (Fallback flow): The alternative flow users experience when the bot fails to understand their message.

Conversation Settings​

Fine-tune the conversational behavior of your bot to align with your desired user experience. The following are the various conversation settings that are available on the Studio page:

Global OptionsDescription
Welcome messageThis is the first message that is automatically sent by the bot to the user. To configure the Welcome message, click Edit icon.

You can enter the text or select the type of message that you wish to display to the user.

Note:Welcome message is supported only for web and app SDK and it is not supported for any of the Channels.
FallbackDisplays a message when the bot is unable to provide the solution to the user's query. To trigger a specific flow when the bot does not understand the user message, click Edit icon and select the flow to be executed from the drop-down.

You can enter the fallback message and select a flow from the Execute Flow drop-down.
FAQsHere, you can add FAQs to your bot by clicking + Add and you can also upload the FAQs in bulk by clicking Upload.
ChannelsYou can add various Channels to your bot based on your requirements.
Document SearchThe bot responds to the user based on the data from the uploaded document. To add a new document, click + Add document.
SuggestionThis can be used when you want to redirect a user when the requested answer is not found.
For example, you have asked a bot, "I want to know my order status for the day". If bot does not understand or not sure about your question, then the relevant suggestions are displayed. 
In these cases, bot replies with a message, "My apologies, I was not able to understand what you have asked." and provides relevant suggestions to the user. Also, it provides an option, "Not Satisfied". When user selects "Not satisfied" option if they are not satisfied with any of the bot suggestions, bot searches for the answer from the Document Search if it is configured; otherwise, a fallback message is displayed.
Suggestion triggers when an utterance doesn't reach minimum confidence. It extracts the similar intents that match any intent and exposes the 20 matches with the highest similarity confidence. It then calls another API internally to extract the descriptions of the journeys and display them.
Note:
If the confidence is between 0.6 and 0.9, then the bot provides suggestions.
If the confidence is below 0.6, then the bot displays a fallback message.
Language settingsYou can add multiple languages to your bot based on your business needs. Click Edit icon to add languages.

Select languages from drop-down and click Save.
CalloutThese banners on top of the chat can be used to promote a brand and/or get visitors from the host website to go to the advertiser's website.

Note: You can add four callouts with two buttons each.
You can choose the type of callout banner, text or image.
In the text banner, you can type the text for the banner and buttons to support it.

In the image banner you can add an image and enable Add link upon image click for the image to redirect the user to a certain flow or URL when clicked.
note

Welcome message is supported only for web and app SDK and it is not supported for any of the Channels.

info

Text can be entered in Bot says field. Images, Carousel, Videos, Files and Quick replies can also be added. When Execute Flow is selected, the start of the flow will be directed to the selected flow.

Preview​

The preview option lets you take a preview of the entire bot, giving you a look at how the bot would look, function, and converse with the end user.

When you click the Preview button, it opens on the right side. You can also click the arrow on the top to open the bot on a different page and test it.

Publish changes​

Publish changes has the following options:

Publish bot​

Publishes the entire bot live. Click here to learn more about publish.

Export as bot template​

Export the bot as a template to be reused in the future. Flows, databases and FAQs of the bot can be exported.

To export a bot template:

  1. Click Export as bot template.
  2. To set up export, fill the following fields:
NameDescription
Template nameName of the template. This is auto-filled and you can modify the name of the template according to your business needs.
OverviewGive a gist of the selected flow, this appears on the template.
Component typeSelect the component type based on the selected.
Template iconSelect a preconfigured template icon or you can upload the icon of your choice.
DescriptionDescribe the template.
FlowsYou can include other flows in the same template by selecting the flows from the dropdown.
Export table dataInclude database tables to your template by selecting the respective database from the dropdown.
Add FAQsTo export FAQs along with your template, simply toggle the corresponding button.
Key featuresYou can include up to three key features in your template, which will be shown on the template cover.
Template tagsTags entered in this field can enhance search results when looking for specific keywords related to your use case.
Use casesYou can select up to three categories that are available in the drop-down that suit your template.
LanguagesTo view your template in multiple languages, you can select the language of your choice.
IndustriesYou can choose up to three industries that match your flow.
ChannelsChoose the channel you wish to integrate your template with.

  1. Click Next.

  2. If you've selected multiple flows, you can switch between them and preview the one you want to export.

  3. Choose who can access your template using these options:

  • Myself: Publish and reuse the template for your own use.

  • Training: Only users with access to the specific subscription can use the exported template.

  • Public: The template is published in the Marketplace and publicly available within the bot after admin approval.

  1. Templates submitted for review can be viewed under User account > Exports.

You can view the status of the template on this page.

  1. Templates exported to a private marketplace are automatically published without review. For subscription exports, they require review and are displayed on this page. The subscription admin can view, test, and publish them. Similarly, templates exported to the public marketplace also undergo review and appear here. The public marketplace admin can view, test, and publish those templates.

  2. After selecting the access option for the template, the system will list the approved templates under the corresponding category.

Welcome (Start Flow)​

This is the flow any user would encounter when they access the bot. Here, you can assign a flow the bot should carry out when a new user uses the bot for the first time.

To set a welcome flow:

  1. Click the Welcome option.
  2. Create a flow.
note

If you had previously set a welcome flow in our previous UI, you will see a legacy node that contains the old welcome flow settings. It cannot be edited.

Any of the flows can be set as a welcome flow.

Fallback (Fallback Flow)​

The bot switches to this flow when it doesn't understand the user input.

To set a fallback flow:

  1. Click the Fallback option.
  2. Create a flow.
note

If you had previously set a fallback flow in our previous UI, you will see a legacy node that contains the old fallback flow settings. It cannot be edited.

Any of the flows can be set as a fallback flow.
